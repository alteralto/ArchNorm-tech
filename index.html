Первый нормативный проход — за минуты, а не за часы

ArchNorm отвечает на вопрос по СП/СНиП и даёт проверяемую ссылку на документ, пункт, страницу и фрагмент-доказательство. Поиск нормы сокращается с ~2 часов до 7–10 минут.

Проблема

Поздно найденная норма стоит бюро миллионы

Архитектор тратит до 2 часов в день на ручной поиск по своду правил. Норму легко пропустить, а ошибка, найденная на экспертизе или стройке, оборачивается переделкой ~5% бюджета объекта. С 01.03.2026 (309-ФЗ) ответственность ГИП ужесточена — цена ошибки выросла. Спрос подтверждён рынком: прямой облачный аналог Norma.AI привлекает инвестиции, во Франции работает ArchiNORM — а ниша on-prem / закрытого контура свободна.

Ручной поиск

Тысячи страниц СП/СНиП, перекрёстные ссылки, версии документов.

Закрытые данные

Как работает

Прозрачный RAG-пайплайн — без «магии» фреймворков

Каждый компонент — отдельный читаемый модуль. Ответ всегда сопровождается источником: без цитаты нормы ответ не выдаётся.

Ingestion
PDF → текст (pypdf) + таблицы (pdfplumber → Markdown), OCR при необходимости
Чанкинг
границы пунктов по структуре СП: раздел/пункт/таблица/примечание + метаданные
Retrieval
embeddings (BGE-M3) + векторный поиск ChromaDB, расширение запроса
Cross-ref
разрешение «см. п. 6.2», «таблица 7» внутри норм
Ответ + источник
LLM формирует ответ с ссылкой: документ, пункт, страница, фрагмент
для технического жюри

Агенты

Мультиагентность

Оркестратор маршрутизирует вопрос к доменным агентам (напр. пожарная безопасность) — расширяемая основа под разделы норм.

VLM

Vision (Qwen3-VL)

Диаграммы, схемы и таблицы СП обрабатываются как изображения — до 4 вложений на вопрос.

Качество

Eval-driven + ~25 тестов

Hit@K / MRR / NDCG на контрольном наборе; pytest-гейт в CI. Анти-галлюцинации: 3 режима ответа + отсечка out-of-corpus.

Docker

Slim-образ

~200 МБ, inference в облаке, без GPU на VM

CI/CD

GitHub Actions

pytest → Yandex Container Registry → SSH-деплой

Security

API-key · TLS · rate-limit

Caddy + security-заголовки, аналитика без PII

Privacy

PII-редакция

Presidio + распознаватели РФ (ИНН/ОГРН/кадастр)

Архитектура

Один код — два контура развёртывания

Ядро RAG общее; контур выбирается переменными окружения, а не форком кода.

Контур 1 · главный

Локальный on-prem (desktop)

Tauri 2 + Python sidecar + llama.cpp (Qwen3-VL-4B) + BGE-M3 + ChromaDB embedded. Полный offline, PII-редакция, Ed25519-лицензирование с offline-активацией по HWID. Данные клиента не покидают контур бюро.

Контур 2 · облако

Yandex Cloud (демо + SaaS)

FastAPI за Caddy (TLS) в Docker, YandexGPT как LLM и embeddings, ChromaDB в volume, Container Registry + CI-деплой. Demo-корпус без NDA-документов, доступ по API-ключу.
